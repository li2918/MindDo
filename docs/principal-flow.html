User Flow

校长 · Principal

单校区全权管理者：能看到本校区所有功能与真实金额，能批退款 / 报销，需要在教务、财务、市场、运营之间做关键决策。日常以巡检 + 决策 + 上报为主，授权 校区运营 / 市场专员 做具体执行。

校区范围 · 单一 金额 · 真实可见 审批权限 · 全部（含退款） 权限模板 · principal (*)

核心职责 · 校长每天要做的事

01
🩺 校区健康度晨检 每天上午
  • 打开数据中心 / 数据大屏，扫一眼本校区今日核心指标
  • 注意：在读学员数、本月收入、转化率、续费率
  • 识别异常波动（突然下跌、超出预期），追问运营 / 市场
💰 财务详情与审批 每天 · 待办触发
  • 审批 退款 / 报销（独家权限：approve.finance）
  • 查看完整账单 / 工资 / 合同（无金额脱敏）
  • 核对每日订单与流水
📣 市场转化复盘 每周
  • 看渠道效果表 + 漏斗，找出高 / 低转化渠道
  • 评估市场专员的本月业绩
  • 调整下月投放策略 / 推广预算
📚 教务质量监督 每天 / 不定期巡视
  • 看学员名单、班级容量、上座率
  • 检查考勤完成率、识别迟交的教师
  • 处理重大学员投诉 / 续费谈判
🔁 续费谈判与跟进 每周
  • 查看 7 天 / 30 天内到期学员
  • 识别高风险流失学员（出勤率低 / 反馈差）
  • 与家长直接沟通 / 给出特殊优惠决策
👥 员工管理（本校区） 每周 / 季度
  • 查看本校区员工花名册（教师 / 运营 / 市场）
  • 调整员工状态（在职 / 休假 / 离职）
  • 新员工入职填报基础信息
⚙️ 校区业务设置 按需 / 不常改
  • 授权运营调整营业时段 / 试课时段 / 教室 / 公告
  • 重大配置变更亲自把关
📊 月底业绩复盘 每月
  • 导出本月订单、转化、教务、出勤数据
  • 评估各团队（运营 / 市场 / 教师）KPI
  • 向超管 / 总部上报关键数据

子流程 · 退款审批

04

退款 / 报销审批是校长独有的关键权限（campus-ops 看不到）。流程：待办收到 → 校长进入审批 → 决定 → 写回 + audit。

  1. 触发：家长或员工发起退款请求，写入 minddo_approvals，status=pending
  2. 通知：校长侧栏 待办 / 财务 · 审批 出现红色徽章
  3. 查看详情：requester、amount、reason、submittedAt 一目了然
  4. 决策：金额（真实可见）、家长背景、退款理由综合判断
  5. 点击 [批准] / [拒绝]：写入 status=approved/rejected + updatedAt + 审批人 id
  6. Audit：approval.decision 带 before/after
  7. 下游：若批准，财务核心相应退款流程（人工或自动）

其他重要操作

05

📺 数据大屏使用

本校区跨域数据的"指挥中心"视图。

  1. 数据中心 → 数据大屏 → 进入大屏模式
  2. 全屏显示：在读 / 线索 / 试课 / 出勤 / 收入 / 销售排行 / 公告
  3. 右上角时段选择（本月 / 本周 / 全部 / 自定义）
  4. 每 30 秒自动刷新数据
  5. Esc / ✕ / 退出全屏 都关

👥 员工管理（本校区）

查看 + 调整状态，不能编辑全局角色。

  1. 内部管理 → 员工 sub-tab
  2. 看到本校区所有员工（教师 / 运营 / 市场）
  3. 可编辑：状态（active / leave / inactive）、部门、联系方式
  4. 不能：删除员工、改全局角色 ID
  5. 新员工入职：另开员工档案，填写基础信息

🔁 续费跟进决策

7 / 30 天到期学员的优惠 / 谈判决策。

  1. 财务中心 → 续费看板
  2. 识别即将到期的学员，按时间排序
  3. 查看学员历史：上课次数、出勤率、反馈评分
  4. 必要时给特殊优惠（折扣 / 加赠课时）
  5. 追加 audit membership.renew

📊 月底业绩复盘

导出数据，向上汇报。

  1. 每个模块都有 CSV 导出 按钮（线索 / 学员 / 班级 / 订单 / 员工 / 渠道效果）
  2. 导出当月数据，整合成月报
  3. 查看审计日志了解关键决策时间线
  4. 向超管 / 总部汇报亮点 + 异常
