User Flow

校区市场运营 · Campus Marketing

单校区获客与转化型角色：从线索录入、跟进、试课预约到付费转化的全链路负责。本流程图覆盖一个市场专员的标准工作日所有触达的页面、状态机、写入的存储与审计记录。

校区范围 · 单一 金额 · 不可见 审批权限 · 无 权限模板 · campus-marketing

核心职责 · 市场专员每天要做的事

01
🎯 线索跟进 每天若干次 · 高优先级
  • 从「今日待跟进」队列依次联系：📞 拨号 / ✉️ 邮件
  • 每次接触都用 添加跟进 记录（电话 / 微信 / 邮件 / 到访）
  • 跟进结束时设置 下次跟进日期，状态升级（new → contacted → follow → won/lost）
📋 新线索录入 每天若干次
  • 单条录入：右下角 + 新线索 FAB，自动查重（手机/邮箱）
  • 批量导入 CSV / TSV：预览 → 标记重复 → 只写合法行
  • 所有新线索自动归到本校区，状态默认 new
🏷️ 客户标签与画像 每天
  • 打意向标签（高 / 中 / 低 / 需家长决策 / 时间冲突 / 重新联系）
  • 维护联系历史，每条记录带时间、渠道、备注
  • 批量操作：多选 → 改状态 / 加标签 / 标记已联系
🎤 试课预约与跟进 每天
  • 预约前确认：和家长沟通试课时间、地点
  • 试课当天与校区运营对接（接待 / 签到）
  • 试课完成后录入 市场反馈：1-5 星意向 + 付费倾向 + 备注
📊 渠道分析 每周回顾
  • 查看渠道效果表（线索 / 试课 / 付费 / 转化率 / 金额 / 占比）
  • 识别高转化渠道，反馈给上级建议加大投放
  • 关注个人在团队排行榜中的位置
📈 个人业绩追踪 每月 / 目标管理
  • 设置月度目标：线索数 / 试课数 / 转化数 / 业绩金额
  • 看 4 条进度条：实际 vs 目标，颜色显示达标情况
  • 临近月底加大跟进力度，冲刺目标
💬 沟通话术维护 按需 / 不常改
  • 维护沟通模板（试课确认 / 试课提醒 / 试课后跟进 / 重新激活）
  • 记录跟进时 📋 一键插入模板，省去重复打字
  • 必要时根据反馈调整话术
🤝 跨角色协作 持续
  • 与校区运营：试课家庭到场前确认、试课中回访
  • 与校长：转化困难线索升级、特殊优惠申请
  • 试课完成后与运营核对市场反馈

子流程 · 线索详情模态框

04

从 CRM 列表任意一行的 详情 按钮触发。整合 客户标签 · 下次跟进日期 · 联系历史 · 状态变更 · 备注 五个独立子模块。

  1. 打开：渲染只读字段网格（学员 / 家长 / 电话 / 邮箱 / 渠道 / 意向 / 校区 / 试课时间 / 进入时间 / 最近联系）
  2. 客户标签：现有标签以 chip 显示，可×删除；右侧 6 个常用建议（高意向 / 中意向 / 低意向 / 需家长决策 / 时间冲突 / 重新联系），自由输入支持回车追加
  3. 下次跟进：date picker。设置后 CRM 行显示 📅 倒计时徽章；今日到期变金色，逾期变红色
  4. 状态 + 备注：5 选 1 状态下拉（new / contacted / follow / won / lost）+ 自由文本备注
  5. 联系历史：滚动列表（最新在上），每条带时间、渠道、备注
  6. 添加跟进：channel select（电话 / 微信 / 邮件 / 到访 / 其他）+ 备注 + 📋 模板插入 → 点击 添加跟进
  7. 保存：写回 minddo_trial_leads，追加 lead.update audit，re-render CRM 列表
  8. 标记已联系：快捷键 — 一键 stamp lastContactAt + 状态从 new → contacted

子流程 · 新建线索 + 查重

05

从 FAB 触发。提交时 findDupLead() 扫描已有 minddo_trial_leads，按手机号（去空白 + 小写）和邮箱（trim + 小写）匹配。

  1. FAB 点击：仅 campus-marketing 角色可见
  2. 表单填写：学员 / 家长 / 电话（必填）/ 邮箱 / 渠道 / 意向 / 备注
  3. 提交：先 findDupLead(phone, email)
  4. 有重复 → 警告横幅 ⚠️：「该手机号已存在 / 该邮箱已存在 / 都已存在」+ 3 个动作
    • 查看已有线索 → 关闭新建模态框，打开线索详情模态框
    • 仍然新建 → 强制提交（设 pendingForceSubmit=true）
    • 取消 → 关闭警告，等待用户修改
  5. 无重复 → 写入 minddo_trial_leads，crmStatus="new", trialStatus="scheduled", 当前 ISO 时间
  6. Audit：lead.create

其他高频操作

06

📥 CSV 批量导入

线索清单一次性导入，自动跳过重复。

  1. CRM 头部 📥 批量导入 按钮
  2. 上传 .csv / .tsv 或粘贴文本（首行表头）
  3. 必需列：studentName + phone（支持 学员 / 姓名 / 电话 别名）
  4. 解析后预览表格，每行标记 可导入 / 重复 / 异常
  5. 确认 → 只写合法行，跳过重复
  6. 每条新线索写 audit lead.import

🔢 批量操作

CRM 多选后底部浮动操作条。

  1. 勾选行的 checkbox，或表头「本页全选」
  2. 底部弹出操作条，显示「已选 N 条」
  3. 三种动作：标记已联系 / 改状态（下拉 + 应用） / 添加标签（输入框）
  4. 每个动作都弹确认框防误操作
  5. 逐条写 audit lead.bulk.<action>

📋 沟通模板

常用话术一键插入。

  1. 线索详情 / 添加跟进 行的 📋 按钮
  2. 弹出 popover 列出所有模板（标题 + 预览）
  3. 点击模板 → 内容插入到备注 textarea
  4. 可编辑后保存，原模板不动
  5. 「管理模板…」入口 → 模板管理器（增删）
  6. 首次使用自动播种 4 条（试课确认 / 试课提醒 / 试课后跟进 / 重新激活）

⭐ 试课后市场反馈

试课完成后捕捉市场视角的转化信号。

  1. 试课管理 → 任意 done 状态的行 → 市场反馈 按钮
  2. 1-5 星意向打分（有 hint 文字）
  3. 付费倾向：可成交 / 需再跟进 / 暂不考虑
  4. 自由备注
  5. 保存到 minddo_trial_feedback
  6. 试课行显示「意向 N★」chip（绿/金/红色）
