校区运营 · Campus Operations

单校区现场操作型角色：每天处理上课、试课接待、考勤、请假审批、续费提醒，对接家长、记录交班。本流程图覆盖一个标准工作日内全部触达的页面、状态机、写入的存储与审计记录。

校区范围 · 单一 金额 · 脱敏 ($***) 审批权限 · 仅请假/调课 权限模板 · campus-ops

核心职责 · 校区运营每天要做的事

01
🏫 现场接待与课程 每节课 / 全天滚动
  • 接待今日试课和上课的家庭：签到 → 试课中 / 上课中 → 完成 / 未到
  • 盯今日课程时间轴，识别「进行中」与容量预警（满班 / 近满 / 空班）
  • 协调教师到位、教室准备
✅ 考勤与缺勤跟进 每节课结束 / 每日开班前
  • 逐人录入每节课的出勤（出勤 / 缺勤 / 请假）
  • 必要时调整学员当日上课计划
📨 家校申请审批 待办收到即处理
  • 审批家长发起的 请假 / 调课 申请
  • 查看 reason / 影响课次，决定批准 / 拒绝
  • 退款 / 报销审批由校长处理（看不到）
👥 学员档案维护 按需 / 通常每周几次
  • 更新学员基本信息、年级、Level
  • 暂停 / 退课流程（带原因 + 写入 audit）
  • 跟进 7 天内到期学员的续费
📅 排课与教师值班 每周回顾 / 临时调整
  • 查看周课表、教师值班分布与周课时
  • 识别同教师时段冲突
  • 协调代课 / 调班
⚙️ 校区基础配置 不常改 / 有变动才动
  • 营业时段、试课时段维护
  • 教室列表与容量管理
  • 校区公告发布（临时通知 / 注意事项）
📝 交班与协作 下班前 / 跨班沟通
  • 下班前写一条交班记录给下一班
  • 与市场专员对接试课 → 转化信息
  • 与教务 / 校长上报异常情况
📞 家长沟通 每天若干次
  • 一键拨号缺勤家长（昨日缺勤跟进面板）
  • 试课家庭到场前确认、试课后简短反馈
  • 处理临时变更（请假、改期）

子流程 · 试课接待状态机

04

校区运营在首页"今日试课接待"列表上推动每个家庭的状态。每步会盖时间戳到 lead 记录上，并写入 trial.reception.<action> audit 记录。

初始 已预约 trialStatus=booked
→
[签到]
→
到场 已到场 trialCheckedInAt
→
[开始试课]
→
进行中 试课中 trialStartedAt
→
[完成]
→
完成 已完成 trialEndedAt

任意状态都可点击 [未到] → noshow；最终状态都可 [撤销] → 回到 booked 并清空时间戳。完成会联动调用 flow.markTrialComplete 让市场中心的试课表格也显示"完成"。

子流程 · 学员状态变更

05

在 教务中心 · 学员名单 任意一行点击 改状态 触发。受 students.status 权限保护。

  1. 触发：点击 acad-roster 行的 改状态 链接（仅有权限的用户可见）
  2. 模态框打开：渲染 3 个大选项（active / paused / withdrawn）+ 原因 textarea，预填当前值
  3. 选择新状态：3 选 1，每个选项配 icon + hint（正常上课中 / 暂停-保留学籍 / 退课-不再上课）
  4. 填写原因：自由文本，例如"搬家暂停"、"升学退课"
  5. 保存：
    • 写入 minddo_students 行：status、statusReason、statusChangedAt
    • 追加 audit：kind="student.status"，带 before/after 快照
    • 关闭模态框 → 重新渲染列表
  6. 无变化保护：状态和原因都没动时不写空 audit，提示"状态没有变化"

其他高频操作

06

🎯 考勤录入

从首页"今日课程"行的 [录考勤] 按钮一键跳转。

  1. 选 (offering, classDate)
  2. 显示班级名单 + 3 个状态按钮（出勤 / 缺勤 / 请假）
  3. 逐人或批量打卡
  4. 写入 minddo_attendance，replace-set 语义（同一节课重复录入会覆盖）
  5. 次日的"昨日缺勤跟进"会从这里读取 absent + noshow 行

📨 请假/调课审批

从待办 inbox 处理，受 approve.schedule 权限保护。

  1. 看到 minddo_schedule_requests 里 status="pending" 的行
  2. 读取 type / studentName / targetLabel / reason
  3. 点击 [批准] / [拒绝]
  4. 更新 status → approved/rejected，写 updatedAt + 审批人 id
  5. ⚠️ 退款 / 报销审批走 approve.finance，校区运营看不到

📞 缺勤跟进

"昨日缺勤"面板上一键拨号。

  1. 页面打开自动查询 minddo_attendance 中 classDate=yesterday 且 status=absent/noshow 的行
  2. 用 studentId 反查学员档案拿 phone
  3. 渲染 [联系家长] 按钮，本质是 <a href="tel:...">
  4. 点击直接拉起拨号（桌面端 → 系统默认；手机 → 拨号器）

📝 交班记录

下班前写一条给下一班同事的提醒。

  1. 面板下方 textarea 输入内容
  2. 点击 保存交班
  3. 追加到 minddo_shift_notes 数组：{ id, campus, by, atIso, body }
  4. 列表自动显示最近 6 条，按时间倒序
  5. 权限：shift.write。没权限的角色只能读，看不到 textarea

权限矩阵 · 哪些能看 / 不能看

07
模块 校区运营 同事的权限（对比） 说明
校区运营首页 ✓ 专属 仅自己看到 角色切换时通过 applyRoleSpecificOverview 显示 #opsOverview

待办审批 · 退款 ✗ 仅校长 / 超管 approve.finance —— 校区运营拿不到
财务订单 / 续费 ◐ 金额脱敏 校长 / 超管看真金额 finance.view + finance.renewals；safeMoney() 把 $ 显示成 ***
财务 · 账单/工资/合同 ✗ 校长 / 超管 子页直接隐藏（finance.detail / payroll / contracts）

交班记录 ✓ 校长 / 超管也有 shift.write
