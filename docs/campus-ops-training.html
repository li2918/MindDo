1. 登录与切换身份

01 / Login

1.1 首次登录：从 0 到能看到 dashboard

当前 demo 阶段没有真正的密码登录，使用「切换身份」选择你的员工档案即可。系统会把所选员工 id 写入浏览器的 localStorage（key 名 minddo_active_ops_user），后续每次打开 dashboard 都自动恢复成你这个身份。

  1. 打开 dashboard。浏览器输入网址（或直接点 运营 Dashboard）。如果是第一次打开，系统会自动给你 seed 一份 demo 数据（学员、班级、课次、付费记录等几十条），花 1-2 秒。
  2. 找到右上角的用户菜单。顶栏最右侧有一个圆形头像 + 名字 + 角色 chip 的组合（比如「Lily Hsu · campus-ops」）。这是你当前的身份。如果是第一次，系统默认登录的是最高权限员工，不一定是你。
  3. 点头像/名字，弹出菜单。菜单里有三项：「切换身份」「切换语言」「退出」。点 切换身份。
  4. 在身份列表里找自己。列表按校区分组，每行显示员工姓名、角色（如「校区运营 campus-ops」）、所属校区。找你自己那一行点击 — 注意，市场/校长/超管会有不同的列表颜色 chip，校区运营是橘色。
  5. 页面自动 reload。侧边栏会重新计算你能看到的 tab（campus-ops 看不到工资、退款审批等），看板数据也按你的校区过滤。
  6. 验证成功的标志：右上角 chip 显示「<你的姓名> · campus-ops · <校区名>」；侧边栏里没有「财务详情」「角色管理」「招商加盟」这些超管/校长才能看到的入口。

1.2 每天上班怎么打开

第一次切换身份后，下次打开 dashboard 不需要重新选 — 只要还是同一个浏览器 + 没清缓存，系统会自动恢复成你上次选的身份。

  • 书签 / 桌面快捷方式直接打开 dashboard.html?tab=overview，落地直接是你的首页。
  • 如果发现头像不是你了 — 重复上面 1.1 的步骤 3-5 切回来。
  • 如果 dashboard 完全打不开（空白页 / 报错），可能是 demo 数据损坏。按 F12 打开浏览器控制台 → Application → Local Storage → 找到 localhost 那一项 → 右键 Clear，然后刷新 — 系统会重新 seed。注意这会清空你的本地修改。

1.3 校区怎么选

顶栏会显示「当前校区看板：<校区名>」下拉框。campus-ops 角色只能看到自己所属校区，但下拉框里仍有「全部校区（汇总）」可以临时切换：

  1. 默认显示你的校区。系统从你的员工档案里读 campus 字段，自动锁定到那个校区。
  2. 临时切到「全部校区（汇总）」。用于横向对比（比如「我们校区的留存率比尔湾校区高多少」）。注意：你只是看聚合数据，不能改其他校区的任何东西，写操作都会被权限层拦下来。
  3. 切回自己的校区。从下拉框再选一次自己的校区。系统会记住你最后选的校区写入 minddo_dashboard_campus，下次开 dashboard 还是这个。

1.4 切换语言（中 / EN）

右上角的「中 / EN」chip 一键切换。整个界面文案、表头、按钮都会变。

  • 切换后会写入 minddo_lang，下次自动恢复。
  • 数据本身（学员姓名、备注等）不会被翻译，只 UI 标签会变。
  • 家长端 / 销售端页面用同一个 key，统一切换。
注意 · 别在公共电脑上忘了切回去 用完别人的电脑 demo，记得切回原账号或者关浏览器隐身窗口 — 否则下一个人开 dashboard 会以为是你的身份在操作。

2. 认识界面 · 侧边栏导航

02 / Layout

左侧侧边栏是主导航。每个一级 tab 下有自己的子菜单。下表列出你作为校区运营能看到的所有 tab，以及一键直达入口：

一级 Tab 用途 直达
🏠 首页 校区运营专属首页：今日重点、待办、课程时间轴、容量预警。 打开
📥 待办 请假、改期、合同到期、试课待评估 — 统一处理面板。 打开
📣 市场中心 只读 CRM：看本校区试课线索、各渠道效果。不写入。 打开
📘 教务中心 核心工作区：学员、班级、排课、考勤。 打开
💰 财务（脱敏） 能看付费记录和续费提醒，金额显示为 $***。不能看详单 / 退款 / 工资。 打开
📈 数据中心 各维度的运营分析。财务图表对你脱敏。 打开
🛠 内部管理 本校区员工、教师列表（只读）+ 操作日志。 打开

3. 每日例行清单

03 / Daily

每天的工作可以分成 4 个时段。下面列出每段时间具体要做什么、在哪里做、做完该看到什么状态。建议把这一节当 checklist 用，每条都打钩再下班。

3.1 到岗后（约 8:30 - 9:00）

目标：心里有当天的全局图，没有意外要爆。

  1. 打开 首页。顶部三块卡片立刻告诉你三件事：① 今天有几节课、几个试课家庭；② 待办总数；③ 关键预警（设备故障 / 满班警告 / 临期合同）。如果待办总数 = 0 且预警条没出现 = 安全开工。
  2. 扫一遍「课程时间轴」面板。横向时间条按 30 分钟为格，鼠标悬停在每节课上能看到「班级名 / 教师 / 教室 / 学员数」气泡。注意 红色边框 = 容量超员，黄色边框 = 教师未确认，要立刻找教师确认。
  3. 处理隔夜累积的待办。到 待办，逐条通过 / 驳回。家长一般 6-9 点提交请假最多。详细审批方法见第 6 节。
  4. 核对今天的试课家庭。到 市场 · 试课管理，把当天的家庭都展开看一遍：教练知道几点接谁吗？有特殊需求（孩子怕生 / 家长要求观摩）没？把这些信息当面同步给试课教师。
  5. 设备 + 教室。到 业务设置 · 教室 看本校区教室状态。任何教室是「待维修」就把今天用到它的班级提前调走。

3.2 课程期间（滚动）

目标：实时反应每个突发状况，让家长无感切换。

  1. 家长进门 = 签到。试课家庭：到 试课管理 在那一行点「签到」。正式课学员：通常由教师在课堂里自助签到，你不用管。
  2. 盯「课程时间轴」的颜色变化。从开课到结束颜色会从灰→蓝（进行中）→绿（完成）。如果某节课到点了还没变蓝，可能是教师忘记开签到 — 打电话或微信问一下。
  3. 家长来电请假。立刻代录 — 详见第 6 节「代录请假」。不要说「您去 App 自己提交吧」让家长再操作，体验很差。
  4. 孩子迟到 / 中途离开。等课结束时录考勤的时候反映出来，状态选「迟到」。
  5. 临时调课。家长说今天来不了想换到周六同时段：先到 教务 · 排课 看目标时段有没有空位，再回审批表帮家长改期。

3.3 每节课结束后 15 分钟内

目标：考勤当节闭环，不带未结记录过夜。

  1. 录正式课考勤。到 教务 · 考勤 → 选日期 + 班级，逐人打勾。详见第 5 节。
  2. 试课家庭收尾。到 试课管理 点「完成」或「未到」。完成的需要立刻填评估表 — 趁记忆新鲜质量最高。详见第 4 节。
  3. 更新作业 / 成长记录（可选）。如果教师把作业反馈给你了，到学员详情→作业 tab 录入。一般教师自己录，你只在教师没空时代录。

3.4 下班前（约 18:00 - 19:00）

目标：把今天的状态清清白白交给下一班同事。

  1. 清空待办收件箱。再扫一遍 待办。理想状态：总数 = 0。如果有处理不完的（比如等家长回复），别强行通过，留在那让明天的人继续。
  2. 跟进续费。到 财务 · 续费提醒，看「剩余课时 ≤ 3」的红色行 — 给这些家长发一条续费提醒微信。详见第 7 节。
  3. 写交班记录。到 首页 底部「交班记录」面板。三栏：今日完成 / 遗留 / 明日重点。详见第 9 节 — 这是必填项。
  4. 检查明日提醒。「明日重点」栏里写明天 8 点要先做什么（比如「张老师 9 点的课要提前 30 分钟到，他生病」）。
  5. 下班 ≠ 关电脑就走。把 dashboard 锁屏（Ctrl+L 或 Win+L），下次同事接班时直接接上。
🚦 红绿信号灯心法
  • 🟢 绿灯：首页待办 = 0，时间轴全绿，预警条空。可以喝口水。
  • 🟡 黄灯：有 1-2 个待办或预警。10 分钟内处理完。
  • 🔴 红灯：>3 个待办 / 预警，或时间轴有红色超员格。停下手头的事先扫红色。

4. 试课接待

04 / Trial

试课线索由市场团队录入（你不能改），但你负责现场接待 + 状态推进 + 评估反馈。试课家庭的体验决定了 70% 的转化率，这一节是 campus-ops 最关键的日常。

状态机：booked（预约）→ confirmed（已签到）→ in_progress（试课中）→ completed（完成）或 noshow（未到）。所有状态写入 minddo_trial_leads。

4.1 试课前 24 小时：确认

  1. 到 试课管理。顶部切「日期」筛选选明天，看明天的所有预约。
  2. 逐家庭看预约信息。每行展开后能看到：家长姓名 / 电话 / 微信、孩子姓名 / 年龄 / 年级、报名科目、特殊需求、来源渠道。
  3. 给家长发提醒。点行尾的「联系家长」按钮，弹出联系卡片（电话 / 微信 / 邮箱）。不要直接打电话 — 先用微信发一句话提醒（系统有模板：「您好，明天 X 点 X 的试课课程已为您预留，请按时到 [校区地址]。如有变动请提前回复，谢谢！」）。
  4. 登记到课前清单。明天值班的人会看到这条「已联系确认」标签，避免重复打扰。

4.2 试课当天到店：签到

  1. 家长进门，先问候。对小朋友：「你是 X 同学吧？今天我们要上 [科目]，你听过这个吗？」对家长：「您是 X 妈妈/爸爸？请坐，我先给孩子简单介绍一下今天的安排。」
  2. 到 试课管理，找到那一行，点「签到」按钮。状态从 booked 切到 confirmed。系统记录 arrivedAt 时间戳。
  3. 给家长一份课前问卷。纸质或扫码二维码都行 — 问「孩子对什么内容最感兴趣」「目标」「过往经历」。这些后面填评估表会用到。
  4. 引导到教室。告诉家长「您可以在外面等候区/观摩区休息，60 分钟后我们一起跟您反馈孩子的表现」。不要放任家长在教室门外走动，影响孩子专注。
  5. 点「开始试课」按钮。状态切到 in_progress。

4.3 试课中：观察 + 记笔记

试课中你的主要工作是静默观察。建议在记事本上记三件事：

  • 兴趣度：孩子主动提问 / 笑 / 兴奋的瞬间次数。1-5 分。
  • 专注度：能持续多久不被周围干扰。< 5 分钟 = 1，> 30 分钟连续 = 5。
  • 反应速度：教师问问题后多久能回答 / 动手。

这些不是给孩子打分 — 是给后续推荐课程做参考。

4.4 试课结束：填评估表 + 反馈

  1. 课程结束时到 试课管理，点该行的「完成」按钮。弹出评估表单。
  2. 填评估表（字段清单）：
    • 兴趣度：1-5 滑杆。结合 4.3 你的笔记。
    • 专注度：1-5 滑杆。
    • 推荐等级：Beginner / Intermediate / Advanced — 这是给市场和教学排班用的。
    • 建议课程：从下拉选一门或多门最匹配的（系统会按推荐等级过滤候选）。
    • 文字评价（中英双语）：写一段 100-200 字的描述。家长会看，所以语气要积极 + 具体。差的方面用「需要更多时间适应」「未来可以加强 X」等委婉表达。
  3. 用 AI 草稿省时间。表单顶部「✨ AI 评估草稿」按钮 — 先在「观察」框里写 3-5 句你的笔记（孩子表现、教师反馈、家长态度），点 AI 按钮，几秒后生成一段格式完整的草稿。点「应用到评估表」直接填进各字段，再手动微调。
  4. 提交。状态切到 completed，市场团队会在他们的 CRM 看到这条评估，3 天内联系家长推荐套餐。
  5. 当面跟家长反馈。把评估表「文字评价」的中文版打印或念出来给家长听。不要当面提价格 — 价格是市场的事。

4.5 未到（noshow）的处理

  1. 预约时间过了 15 分钟还没到，先打电话或微信问一下情况。系统会自动在 30 分钟后标黄提醒你。
  2. 家长说今天来不了想改期。到行尾「改期」按钮，选新时段。状态保留 booked，不算 noshow。
  3. 家长完全联系不上。等到当天晚上 18:00 还没消息 → 标 noshow，备注栏写「未联系上」。市场会判定是否再约。
评估表小窍门
  • 「文字评价」是家长能看到的，写之前先在脑子里过一遍「我会怎么读这段给孩子听」。
  • 「建议课程」最多选 2 门 — 选太多市场会觉得你没主见。
  • 评估提交后不能改，所以仔细审一遍再提交。

5. 考勤录入

05 / Attendance

正式课的考勤是 campus-ops 最高频的录入操作 — 每节课结束都要做一次。考勤数据进 minddo_attendance，三个地方会引用：① 学员详情 → 出勤 tab；② 续费提醒（出勤率太低会标黄）；③ 财务结算（教师课时酬金按完成的课次算）。所以录得准、录得快很重要。

5.1 状态定义

状态含义影响
出勤正常到课。消耗 1 节课时，教师可结算。
迟到到课但迟到 ≥ 15 分钟。消耗 1 节课时；备注栏要写「迟到 X 分钟」。
缺勤未到且未请假。消耗 1 节课时；自动生成「缺勤提醒」push 给家长。
请假家长提前申请且已审批通过。不消耗课时；如需补课需走「调课」流程。

5.2 标准录入流程

  1. 打开 教务 · 考勤。默认显示「今日」+「全部班级」。
  2. 选课次。顶部三联下拉：日期 → 班级 → 上课时段。选刚结束那一节。如果是连续上的两节同班，每节单独录。
  3. 看花名册。系统列出该班全部学员（按姓氏拼音排序）。每行：头像 / 姓名 / 当前级别 / 状态选择器 / 备注框。
  4. 逐人勾选状态。从下拉选 出勤 / 迟到 / 缺勤 / 请假。批量操作：顶部「全部出勤」一键设置全班出勤，然后只改异常的几个。
  5. 异常状态必填备注。
    • 迟到：备注「迟到 X 分钟，[原因]」（如「迟到 20 分钟，校车延误」）。
    • 缺勤：备注「未请假未到，已联系家长 [结果]」。
    • 请假：选「请假来源」 — 家长 App 提交（自动关联审批记录）/ 家长当天电话（你手动代录）/ 教师确认（孩子生病当天通知）。
  6. 点「保存全部」。系统逐行写入 minddo_attendance，每行一条审计记录。完成后顶部弹出「✅ 已保存 X 条记录」提示。
  7. 验证保存成功的方法：到任意一个学员的详情抽屉 → 出勤 tab，最新一行应该就是你刚录的。

5.3 录错怎么改

考勤是「事实记录」，错了一定要改 — 否则财务对账会出问题。改法：

  1. 到同一个 考勤面板，重新选当天 + 那个班。
  2. 找到那个学员，重新选状态 + 改备注。
  3. 保存。系统不会覆盖原记录，而是写一条「变更」记录到审计日志，旧值 + 新值 + 谁改的 + 什么时间都保留。
  4. 当事人（教师 / 家长 / 校长）如果对修改有异议，去学员详情 → 变更历史 tab 看 diff。

5.4 常见场景

  • 整个班级全部出勤：顶部「全部出勤」一键勾选 → 保存。30 秒内搞定。
  • 某学员调到别的班了：到 教务 · 排课 先调整学员所属班级，再录今天的考勤 — 否则那个学员还在原班的花名册里。
  • 新插班学员第一节课：花名册里没有他 — 到 教务 · 学员 把他加入这个班，再回考勤面板录入。
  • 试课家庭混到正式课：不要在考勤里录他 — 试课考勤在试课管理里走。
注意 · 不要漏录、不要批量乱录
  • 1 周内未录入的课次，财务面板会自动标红 — 越拖越难补（要回忆当时的细节）。
  • 不要为了快「全部出勤」之后不仔细看，有一个缺勤被错记成出勤，家长发现会很不爽。
  • 每节课结束 15 分钟内录完是金标准。

6. 待办处理：请假 / 改期 / 合同 / 试课评估

06 / Inbox

「待办」是 campus-ops 的主战场。所有需要你今天做决定的事都集中在这里。家长在 App 提交的请假 / 改期、合同到期提醒、试课评估缺失 — 全在一个面板里统一处理。

6.1 待办面板的结构

打开 待办 后看到的：

  • 顶部 KPI 卡：待办总数 / 请假改期 / 财务审批 / 合同到期 — 一眼看到各类积压。
  • 过滤 pill：「全部 / 请假改期 / 退费报销 / 合同到期 / 试听评估」。先点你最熟的那一类，集中处理。
  • 卡片列表：每条一行，展示 [类型] + [学员姓名 + 课次摘要] + [家长备注] + [时间] + [操作按钮]。

6.2 类型 1 · 请假 / 改期（你能审批的）

家长在 App 提交「我家小孩明天的课请假」后，会到你这里来。处理流程：

  1. 顶部过滤选「请假改期」。列表只剩这一类。
  2. 点卡片展开。能看到：申请类型（请假 / 改期 / 退课）、原课次（日期 + 班级 + 时段）、目标课次（如果是改期，家长选的新时段）、理由（家长填的备注）、提交时间。
  3. 判断要不要通过 — 三个原则：
    • 提前 24 小时以上 → 一般通过。
    • 当天临时请假 → 看理由，生病 / 急事一般通过；「想去玩」可以委婉沟通后通过。
    • 已经请过 3 次以上 → 通过前给家长打电话沟通学习节奏，避免影响进度。
  4. 点「✓ 通过」。状态机：请假 → 该学员当节考勤自动标「请假」；改期 → 原课次自动出 + 目标课次自动入。系统给家长 App push 一条「您的申请已通过」。
  5. 点「✗ 驳回」。弹出理由输入框 — 必填。建议模板：「您好，本次申请未能通过：[具体原因]。如有疑问请回复或联系校区老师。」理由会原文 push 给家长。
  6. 审批后这条从待办移除。但会保留在学员详情 → 申请 tab 的历史里。

6.3 类型 2 · 财务审批（只读 — 转给校长）

退费、报销这类需要校长（principal）权限。campus-ops 能看到但不能批准。

  1. 过滤选「退费报销」，卡片右上角会显示 🔒 只读 标签。
  2. 点开看：金额（脱敏为 $***）、申请人、理由。
  3. 判断紧急程度，紧急的：直接微信 @ 校长，把卡片右下角「分享给校长」按钮的链接发过去。校长一登 dashboard 就能直接定位到这条。
  4. 非紧急：什么都不用做，校长每天会自己扫一遍。

6.4 类型 3 · 合同到期提醒

合同还有 30 天到期 / 已过期未续签 会自动出现在待办。

  1. 过滤选「合同到期」。
  2. 点「前往原页面」按钮跳到 续费提醒 该学员那行，看完整的续费状态（剩余课时 / 上次联系时间 / 历史付费金额）。
  3. 给家长发续费提醒微信（详见第 7 节「续费跟进」）。
  4. 沟通完成后到 学员详情 → 变更历史 加一条备注，这条会自动从待办消失。

6.5 类型 4 · 试课待评估

试课家庭课已经上完但评估表还没填，超过 24 小时会自动进待办。

  1. 过滤选「试听评估」。
  2. 点「前往原页面」直接打开 试课管理 该家庭的评估表单。
  3. 按第 4 节的流程填评估，提交后从待办消失。

6.6 代家长录入请假（家长不会用 App 时）

家长来电话临时请假，但 App 不会用 — 你需要代他录：

  1. 打开 教务 · 学员，找到那个学员。
  2. 点学员姓名打开详情抽屉 → 切到「申请」tab。
  3. 右上角「+ 新增申请」按钮，选「请假」类型。
  4. 填表：申请课次 / 理由（写「家长电话申请 [yyyy-mm-dd] [hh:mm]」）/ 联系记录方式。
  5. 提交。这条会立刻出现在你自己的待办里。
  6. 立刻通过（既然你刚帮家长录的，肯定 OK）。完成后告诉家长「已为您操作完成」。

6.7 批量审批小技巧

  • 同一家长一天多条申请：列表会自动聚合，点「按家长聚合」开关，方便统一处理。
  • 典型的「正常」申请（24 小时前提交 + 理由是生病 / 出差等常见原因）：1-2 秒一条点完，10 条 30 秒搞定。
  • 有疑问的：先放着不点，处理完简单的再回来。
审批记录的查询 所有审批都写入 minddo_audit_log。校长 / 超管要查「上周校区有多少请假被驳回」可以到内部管理 → 操作日志，按 kind = "request.decide" 过滤。

7. 续费提醒与跟进

07 / Renewals

「续费提醒」是家长留存率的核心抓手。campus-ops 看不到具体金额（脱敏为 $***），但能看到所有「快用完课时 / 套餐快到期」的学员名单。每个家长被提醒一次的成本远低于一次重新获客，所以这一节每天都要花 15-20 分钟。

7.1 列表怎么读

  1. 打开 财务 · 续费提醒。
  2. 默认排序：按「剩余课时」升序，最少的排最上。同时按「上次联系日期」二次排序，让长时间没联系的先冒头。
  3. 颜色编码：
    • 🔴 红色 = 剩余 ≤ 3 节，极临期 — 今天必须联系。
    • 🟡 黄色 = 剩余 4-8 节，临期 — 本周内联系。
    • 🟢 绿色 = 剩余 ≥ 9 节，正常 — 不用主动催。
  4. 每行展示：学员姓名 + 头像 / 当前级别 / 剩余课时 / 上次付费日期（金额脱敏）/ 上次联系日期 / 出勤率 / 联系按钮。

7.2 跟进单个家长 — 完整流程

  1. 选目标。从红色行开始，点学员姓名打开详情抽屉。
  2. 做功课，先看四项数据（在抽屉的各 tab 里）：
    • 课时使用率（概览 tab）：用了多少 / 总数。低于 60% 说明家长可能犹豫。
    • 近 4 周出勤率（出勤 tab）：< 75% 是预警 — 跟进时先关心「最近怎么常缺课」。
    • 近 2 个月成长曲线（成长 tab）：上升的可以推「保持势头」；持平的可以推「换更进阶的课程」。
    • 历次反馈（反馈 tab）：家长之前抱怨过什么 — 跟进时先回应这些。
  3. 点「联系家长」按钮。位于抽屉右上角或行尾。弹出联系卡片（不是直接拨号）：显示电话 / 微信 ID / 邮箱 / 家长偏好沟通方式 / 最佳联系时间。
  4. 选合适渠道。
    • 微信（推荐）：点「复制微信 ID」→ 切到微信 → 粘贴搜索 → 发消息。
    • 电话：紧急或微信半天没回时用。点「拨号」会调出 tel: 链接（手机端 / 桌面有拨号软件时直接拨）。
    • 邮箱：备用，国内家长一般不查。
  5. 沟通话术参考（系统里有模板，「联系家长」面板下方「话术模板」可以调出）：
    • 剩 3 节：「X 妈妈您好，X 同学的 [课程] 还剩 3 节，下周可能就上完了。给您看一下他这两个月的成长曲线 [发图]，[具体进步点]。下学期想继续跟着学吗？我把可选套餐发您看下？」
    • 剩 1 节 + 出勤率低：「X 妈妈，X 上周缺了 2 节课，最近孩子是不是太忙了？我们可以一起看看怎么调整时间表。」
  6. 沟通后必须留记录。回到学员详情 → 变更历史 tab → 点右上「+ 添加备注」。填：联系渠道 / 关键沟通点 / 家长反应 / 下次跟进时间。这条会自动写审计日志，系统会按「下次跟进时间」给你 push 提醒。
  7. 关闭抽屉。列表里该学员的「上次联系日期」自动更新为今天，颜色不变（剩余课时没变），但下次默认排序会往后挪。

7.3 批量续费节奏（每周一次）

  1. 每周一上午先扫一遍所有红色行 — 优先级最高的 5-10 个家长，2 小时内联系完。
  2. 周三再扫黄色 — 量大但不急，可以发批量微信群发或者模板邮件。
  3. 周五前所有「下次跟进时间已到」的提醒清零。

7.4 续费成功之后

家长口头同意续费后，不是你做收款 — 这部分由市场或校长操作。你的工作：

  1. 到学员详情 → 变更历史 加一条「家长同意续 X 套餐，建议套餐 [选项]」备注。
  2. @ 市场或校长（在备注 @ 他们的姓名），他们会跟进收款。
  3. 收款成功后系统自动把这位学员从「续费提醒」列表移除。
不要骚扰家长 系统会自动屏蔽「7 天内已联系过」的家长不让你重复联系。如果你强行跳过，备注里会被标红「过度联系」— 校长每周会扫一遍。续费靠的是恰到好处的提醒，不是连环 call。

8. 学员信息查询

08 / Students

学员详情抽屉是 dashboard 里用得最多的视图。你接电话、写交班、做续费 — 几乎每个场景都会先打开学员详情看一眼。

8.1 三种打开方式

  1. 从任意列表点姓名。考勤面板、续费提醒、试课管理、待办里所有出现学员姓名的地方都是可点击的链接。
  2. 全局搜索（最快）。任意页面按 Ctrl/⌘ + K 唤起搜索框，输入姓名 / 学号 / 电话 / 邮箱任意一个，列表里点回车直接打开。
  3. 从 教务 · 学员。完整花名册，可以按级别 / 班级 / 校区过滤。

8.2 抽屉的 9 个 tab — 各自看什么

Tab看什么什么时候用
📋 概览 基本资料（姓名 / 年龄 / 年级 / 家长信息）、当前级别、报名套餐、剩余课时。 接电话第一眼看 — 30 秒了解整体情况。
✅ 出勤 历次出勤记录，按月分组，可筛选「缺勤 / 请假 / 迟到」异常。底部有出勤率统计。 家长问「孩子上了多少节课」「最近为什么经常缺」时用。
📝 作业 课后作业列表 — 题目、提交时间、教师评分、未提交标记。 家长问学习情况；教师反馈孩子作业问题。
📈 成长 每月「兴趣度 / 编程能力 / 表达 / 独立性」四维曲线。点曲线某点能看到当月评语。 续费跟进、学期报告、家长会前的功课。
💰 付费 历次付费记录（金额脱敏为 $***）、套餐、生效 / 失效日期。 家长问「我买的什么套餐？还剩多少？」
💬 反馈 家长 / 教师双向反馈历史。每条 + 时间 + 提交人 + 内容。 跟进前看家长之前抱怨过什么，避免重复踩坑。
📨 申请 该学员的所有请假 / 改期 / 退课申请，含状态（通过 / 驳回）+ 理由。 核对家长说的「我之前请过假」是否属实。
🎨 作品集 项目作品（代码 / 截图 / 视频 / 演示链接）+ 学期报告 PDF。 家长问「孩子做出什么了」；招生时展示给意向家长。
📜 变更历史 该学员相关的所有操作日志：谁、什么时间、改了什么字段、前后值。 核对争议（「我没说过这话」「这条改过」）；写交班记录前回顾今日改动。

8.3 常见场景操作

  1. 家长来电问「我家孩子上学期表现」。
    1. Ctrl+K 搜姓名 → 打开抽屉。
    2. 概览 → 看当前级别 + 剩余课时。
    3. 成长 → 看四维曲线，挑 1-2 个上升明显的维度作为亮点。
    4. 反馈 → 拿出 1-2 条教师近期正向反馈念给家长听。
    5. 挂电话后到变更历史加备注「[日期] [家长姓名] 来电询问表现，已用 X / Y / Z 答复」。
  2. 排查考勤异常。家长投诉「我孩子明明来了你们记成缺勤」：
    1. 打开抽屉 → 出勤 tab → 找到争议日期。
    2. 点该行查看「录入人 / 录入时间」+ 备注。
    3. 跟教师核对 → 如果确实错了，到 考勤面板 改正（会留 diff 在变更历史）。
    4. 到变更历史加备注解释。
  3. 试课家庭转正式学员。
    1. 该家庭在 试课管理 列表。
    2. 右上「转正式学员」按钮 → 弹出表单：选班级、套餐（金额脱敏看不到具体数字但选项还在）、入学日期。
    3. 提交后该家庭从试课列表迁出，新学员档案出现在 教务 · 学员。
  4. 更新学员资料（家长换了电话 / 改地址）。
    1. 打开抽屉 → 概览 tab → 顶部「编辑」按钮（如果没有说明你没这个权限，找校长）。
    2. 改完保存，旧值会被记录到变更历史里。
不要靠记忆，靠抽屉 家长打电话进来前，先开学员抽屉再接电话 — 30 秒看一眼，胜过靠脑子翻记忆。

9. 交班记录

09 / Shift

交班是 campus-ops 独有的责任。校长每周扫一次，但下一班的同事每天接班都看 — 信息断了，家长第二天来了就会被「你们怎么不知道我上周说过」打脸。这一节每天必填，不写不下班。

9.1 三栏的填法

面板在 首页 滚动到底部，标题「交班记录」。三栏分别要填：

栏目填什么填法示例
✅ 今日完成 客观数字 + 1-2 个亮点事件。AI 会从审计日志自动汇总数字，你只补亮点。 「上课 8 节、试课 3 个、审批 5 条。亮点：林同学第一次写出完整的 if-else 逻辑，家长很激动。」
⚠️ 遗留问题 没处理完的事 + 已经联系但等家长回复的事 + 设备故障等。每条要可执行。 「① 周妈妈昨天微信问续费套餐，等她回复（已发详情）。② 教室 B 投影偶发花屏，明早维修师傅 10 点到。」
🎯 明日重点 明天第一件要做的事 + 接班人特别要注意的。 「明早 8:30 优先：① 张老师 9 点的课要早 30 分钟到，他生病；② 9:30 试课家庭 王同学第二次来，他妈妈关心进度，准备一份成长曲线打印件。」

9.2 标准填写流程

  1. 17:30 左右开始写。太早会漏掉下班前的事；太晚同事接班看不到。
  2. 点「✨ AI 生成草稿」按钮。AI 会读今天的审计日志（你处理的考勤、审批、续费跟进），自动汇总「今日完成」栏。10 秒出草稿。
  3. 检查 AI 草稿。AI 只能看到系统里的事，「林同学第一次写出 if-else」这类教室里发生的事它不知道 — 你手动补 1-2 句。
  4. 「遗留问题」自己想。过一下今天的微信 / 电话记录，把没闭环的事列出来。每条要写清楚：① 谁的事 ② 现在到哪步了 ③ 下一步谁做什么。
  5. 「明日重点」按重要度排。最重要的写最上面，最多 3-5 条 — 多了同事看不进去。
  6. 点「保存」。立刻写入 minddo_shift_notes。所有同校区 campus-ops 在自己的 dashboard 首页都能看到。
  7. 验证：关掉 dashboard 再打开，首页「交班记录」面板顶部应该出现「今日 X 时 X 分由 [你的姓名] 提交」标签。

9.3 接班怎么读

  1. 第一件事打开 首页 。「交班记录」面板默认展开最近 3 条。
  2. 从「明日重点」看起，再回看「遗留问题」。「今日完成」浏览即可，不用细看。
  3. 对每条遗留：如果是「等家长回复」，看一下家长是否已经回复了；如果是「设备维修」，确认师傅时间。
  4. 看完后点「✓ 已接班」按钮（在面板右下）。这条会记录到审计日志「<谁> 在 <时间> 接班」。

9.4 反例 — 不要这么写

错误示范问题
「一切正常，无特殊事项」没信息量。"正常" 是判断不是事实 — 改成具体数字或挑一条事件。
「家长 X 投诉了一下，处理完了」下次同家长又投诉时不知道这次怎么处理的。改成「家长 X 投诉 [具体内容]，已 [具体动作]，结果 [结果]」。
「明天没事」真没事也写「明早 8:30 例行检查教室 / 时间轴 / 待办」— 至少给同事一个起点。
AI 帮你写 点「✨ AI 生成草稿」会基于今日审计日志自动总结今日完成栏 — 节省 80% 时间。但「遗留问题」和「明日重点」AI 不会，要你自己想。

10. 校区设置

10 / Settings

「业务设置」tab 是本校区配置的统一入口。这些不是每天改的 — 一周改 1-2 次，但每次改都立刻影响家长端 + 教师端，所以要谨慎。每个修改都会被审计。

10.1 🎤 试课设置

控制家长在 App 端能看到哪些试课时段、每个时段最多预约几家。

  1. 打开 业务设置 · 试课。
  2. 时段开放规则。表格展示每周一到周日各自的开放时段（如「周六 10:00-11:00 / 14:00-15:00」）。点 + 添加新时段。
  3. 单时段容量。每个时段独立设容量（默认 3 家庭/时段）。如果你只有 2 间试课教室，把容量设为 2。
  4. 保存后立刻生效。家长 App 在 5 分钟内会刷新到新规则。本周已经预约满的时段不受影响。

什么时候改：① 新增 / 减少试课教师 ② 节假日临时关闭 ③ 招生旺季临时扩容。

10.2 🕒 营业时间

本校区每周开门 / 关门时段。家长端「营业时间」展示直接读这里。

  1. 打开 业务设置 · 营业时间。
  2. 每天一行 — 周一到周日。每行：开关（开 / 闭）+ 开始时间 + 结束时间。
  3. 修改后点「保存」。家长 App 立刻显示新时段；超出时段的预约请求被拒。
  4. 临时关闭（如节假日）：用「假期模式」按钮，选日期范围 — 比改每天的开关方便。

什么时候改：① 节假日 ② 旺季延长营业 ③ 装修 / 设备维护临时停业。

10.3 🚪 教室管理

本校区所有教室的状态。影响：试课时段容量、正式课排课。

  1. 打开 业务设置 · 教室。
  2. 教室列表。每个教室：名字（如「教室 A」）/ 容量（最多几个学员）/ 设备配置（电脑数量 / 投影 / 白板）/ 状态。
  3. 状态三种：
    • 🟢 启用：正常使用。
    • 🟡 待维修：暂时不能排新课，但今天已经排好的课可以继续上。
    • 🔴 停用：完全不能用，所有已排课次需要手动迁到其他教室。
  4. 标「待维修」要做什么：
    1. 到 教务 · 排课 看本周该教室还有哪些课。
    2. 把这些课手动迁到其他教室（拖拽即可）。
    3. 回设置页把状态切到「待维修」并填备注（如「投影坏了，X 月 X 日维修」）。
  5. 新增教室。「+ 新增教室」按钮，填名字 / 容量 / 设备。提交后立刻可用。

10.4 📢 公告

家长端首页 + App 启动页会显示本校区的公告。可以发节假日通知、活动招募、紧急通知（如停课）。

  1. 打开 业务设置 · 公告。
  2. 「+ 新建公告」。填表：
    • 标题：≤ 20 字，家长端会作为列表项的主显示。
    • 正文：100-300 字，支持简单 markdown（**加粗** / 换行）。
    • 类型：通知 / 活动 / 紧急。紧急会显示红色感叹号图标。
    • 生效日期：起 / 止。过期自动隐藏。
    • 推送：勾选「同时推送 App push 通知」会立刻给家长端 push。
  3. 预览 + 发布。预览按钮看家长端实际效果，确认后点「发布」。
  4. 已发布公告：列表顶部「已激活」标签，点行可以编辑（但已经 push 出去的通知不能撤回）。

常见公告：节假日放假通知、暑假 / 寒假班招募、临时停课、特殊活动（开放日）。

注意 · 不要乱改影响范围
  • 所有设置改动都写审计 — 校长每周看一次，频繁乱改会被问到。
  • 不确定的改动先在「假期模式」/ 草稿状态测试，再正式发布。
  • 不能改其他校区的设置 — 跨校区配置只能由超管做。

12. 常见问题

12 / FAQ
看不到某个 tab 怎么办？

侧边栏会自动隐藏你权限范围外的入口。如果突然某个原来能用的 tab 不见了，检查 ① 你是不是被切换到了其他角色（顶栏右上角看当前身份）；② 是不是切到了别的校区。

金额怎么全是 $***？

这是设计如此 — campus-ops 角色的财务面板对真实金额脱敏。需要查具体数字请联系校长，或者通过「续费列表」+「学员详情」交叉核对（这些视图给你看了课时数 / 套餐名 / 续费状态等关键信息，仅金额数字脱敏）。

家长说他点了请假但我看不到？

① 到 待办 顶部过滤选「请假改期」；② 如果列表里没有，问家长是从家长端 App 提交的还是只是微信告诉你的（后者需要你手动代录）；③ 已审批过的申请会从待办移除，去对应学员详情的「申请」tab 找历史记录。

录错考勤怎么改？

直接在 考勤面板 重新选状态保存即可。系统会写入 变更 而不是覆盖 — 旧值 + 新值都保留在审计日志，谁改的、什么时间一清二楚。学员详情「变更历史」tab 也能看到。

AI 按钮（✨）安全吗？

AI 按钮只产生草稿，不会自动写入。生成结果会出现在按钮下方，需要你确认后点「应用」才生效。可以放心使用，省时间。当前 demo 环境 AI 是本地模拟（不联网），上线后会走真实 LLM 但同样是「先生成、再确认」的流程。

切换语言？

顶栏右上角的「中 / EN」按钮切换。你的偏好会被记住（写入 minddo_lang），下次打开自动恢复。

误操作了能撤销吗？

多数操作不能一键撤销，但 所有操作都有审计，可以根据 操作日志 的 diff 手动改回去。重要操作（如批量改状态）前会有确认弹窗，仔细看一眼再点。

家长打电话要查孩子付了多少钱，但我看到的金额是 $***？

这是设计如此 — campus-ops 不应该直接告诉家长金额。合规做法：

  1. 查 续费提醒 该家长行，看「上次付费日期」和「套餐名称」（这两项你能看）。
  2. 告诉家长「您 X 月 X 日购买的 [套餐名称]，详细金额我帮您让校长 / 财务确认后回复您」。
  3. 立刻 @ 校长（在该家长的学员详情变更历史里加备注 @ 校长），校长会回复家长。
系统反应很慢 / 卡住了怎么办？

① 先看右上角是不是有「⏳ 同步中」标签，等同步完再点；② 没标签但还是卡：刷新（F5）；③ 还卡：F12 打开控制台看 Console 有没有红色错误，如果有截图发给技术支持；④ 极端情况：清缓存重新登（注意会丢失本地未保存的输入）。

有人离职了 / 新人入职，员工档案要怎么调？

campus-ops 不能新增 / 编辑 / 停用员工档案 — 这是校长 / 超管的权限。新员工入职流程：① 校长在内部管理 → 员工里新增 → 分配角色（一般 campus-ops）→ 邮箱通知新员工；② 新员工首次打开 dashboard 用「切换身份」选自己。

更多问题 / 反馈渠道？

页面右下角 ✨ 客服 按钮 → 「AI 聊天」tab 直接问，常见问题秒回；复杂问题切到「联系我们」tab 看微信 / 电话 / 邮箱。或者直接 @ 校长。
